# 티니핑 목소리 수집 조사 메모

## 조사 목적

- 퀴즈 화면의 `목소리 듣기` 버튼에 사용할 짧은 캐릭터 음성 클립 확보 가능성을 확인한다.
- 자동 수집 가능성과 현실적인 운영 방식을 분리해 판단한다.

## 현재 확인한 단서

- 2025-06-09 보도자료/기사 기준, 한국가스안전공사 캠페인 영상에는 `하츄핑, 빛나핑, 초롱핑, 반짝핑` 캐릭터가 생활 속 가스안전 수칙을 알려주는 약 2분 분량 영상이 있고, 공식 유튜브 채널 `티니핑TV`와 공사 공식 유튜브에서 시청 가능하다고 소개된다.
- 2025-10-27 기사 기준, `프린세스 캐치! 티니핑` 첫 에피소드가 유튜브 `티니핑TV`의 `티니핑쇼`를 통해 공개된다고 안내된다.
- 2021-03-03 기사 기준, `티니핑TV`는 본편 외에도 장난감 놀이, 율동, 동요 등 숏폼 콘텐츠를 제공한다고 소개된다.

## 활용 가능한 음성 소스 후보

### 1. 공식 유튜브 채널 `티니핑TV`

가장 유력하다.

- 캐릭터 소개 영상
- 숏폼 클립
- 시즌 티저/오프닝/프로모션
- 교육/캠페인 협업 영상

장점

- 공개 접근성이 높다.
- 캐릭터 단독 대사가 포함된 콘텐츠가 있을 가능성이 높다.
- 시즌별 신규 캐릭터 대응이 쉽다.

주의

- 영상 전체 자동 다운로드와 재배포는 저작권 이슈가 크다.
- 배경음과 효과음이 섞여 있어 자동 분리 품질이 낮을 수 있다.

### 2. 방송 본편/예고편

장점

- 캐릭터별 실제 목소리 확보 가능성이 가장 높다.

주의

- 음성 추출은 가능하더라도 법적/운영적 부담이 더 크다.
- 대사 구간 검출과 배경음 제거 작업이 필요하다.

### 3. 공식 캠페인/브랜드 협업 영상

예시

- 가스안전송 같은 공공기관 협업 영상

장점

- 캐릭터 발화가 비교적 또렷하고 짧게 나올 가능성이 있다.
- 퀴즈용 1~2초 샘플 클립을 뽑기 쉽다.

주의

- 일부 캐릭터만 포함될 수 있다.
- 시즌 편중이 생길 수 있다.

### 4. 완구/앱/프로모션 페이지

장점

- 버튼 클릭형 캐릭터 음성이 있으면 품질이 깔끔할 수 있다.

주의

- 현재 공개 웹페이지에 직접 음성 파일이 노출되는지는 별도 확인이 필요하다.
- 구조가 자주 바뀔 수 있다.

## 현실적인 수집 전략 제안

### 권장 전략 A: 반자동 수집

1. 공식 유튜브/공식 페이지에서 음성 후보 URL 목록을 수집
2. 캐릭터별 후보 구간을 메타데이터로 기록
3. 사람이 짧은 발화만 검수
4. 게임용으로 1~2초 클립만 저장
5. JSON에 `voice` 필드 연결

이 방식이 가장 현실적이다.

이유

- 완전 자동보다 품질이 높다.
- 저작권 검토가 필요할 때 어떤 원본에서 왔는지 추적 가능하다.
- 아이들이 누르는 버튼이므로 잡음이 적고 즉시 재생되는 짧은 클립이 중요하다.

### 전략 B: URL만 관리하고 실제 음성은 외부 재생

1. 게임에는 음성 파일을 직접 넣지 않음
2. `목소리 듣기` 버튼 클릭 시 공식 콘텐츠 링크 열기 또는 임베드 재생

장점

- 파일 저장 부담이 적다.

단점

- 퀴즈 흐름이 끊긴다.
- 정적 HTML의 몰입감이 떨어진다.

### 전략 C: 정식 허가 기반 아카이브

가장 이상적이지만 시간이 걸린다.

- SAMG 또는 권리자와 사용 범위 협의
- 캐릭터별 샘플 음성을 별도 자산으로 정리

## 자동화 가능 작업과 수동 검수 작업

### 자동화 가능

- 후보 영상 URL 목록화
- 페이지/영상 메타데이터 수집
- 로컬 폴더 구조 생성
- 음성 자산 파일명 규칙 생성
- 게임 JSON에 `voice` 경로 연결

### 수동 검수 권장

- 실제 캐릭터 대사 구간 선택
- 배경음이 과한 구간 제외
- 음량 정규화 최종 확인
- 저작권/배포 적합성 판단

## 게임 데이터 필드 제안

```json
{
  "voice": {
    "status": "ready",
    "sourceType": "youtube-campaign",
    "sourceTitle": "가스안전송",
    "sourceUrl": "https://...",
    "clipPath": "assets/audio/voices/hachu-ping-s5-01.mp3",
    "durationSec": 1.8
  }
}
```

`status` 값 예시

- `ready`
- `missing`
- `needs-review`
- `external-only`

## 프로토타입 단계 제안

프로토타입에서는 실제 캐릭터 음성 대신 다음 중 하나를 쓴다.

1. 버튼 비활성화 + `준비 중`
2. 샘플용 로컬 효과음 재생
3. 브라우저 TTS로 임시 안내

권장안은 `1` 또는 `3`이다.

이유

- 실제 캐릭터 음성처럼 오해하게 만들지 않아야 한다.
- 나중에 실제 음성 자산으로 자연스럽게 교체 가능하다.

## 결론

기술적으로는 `공식 유튜브/공식 캠페인 영상 기반 반자동 수집`이 가장 현실적이다.

다만 운영 관점에서는 다음 원칙이 안전하다.

- 원본 영상 전체가 아니라 짧은 게임용 클립만 다룬다.
- 출처 URL과 원본 제목을 메타데이터로 남긴다.
- 완전 자동 추출보다는 수동 검수 단계를 둔다.
