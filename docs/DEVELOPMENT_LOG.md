# 티니핑 퀴즈 서비스 개발 내역서

## 📋 프로젝트 개요
- **프로젝트명**: 캐치! 티니핑 이름 맞히기 퀴즈
- **개발 기간**: 2026.05.03 ~ 2026.06.16
- **기술 스택**: HTML5, CSS3, JavaScript (Vanilla JS), Node.js (Data Crawling), Python (Local Dev Server)
- **배포 환경**: GitHub + Cloudflare Pages (자동 배포)
- **접속 주소**: [bit.ly/tpn-quiz](https://bit.ly/tpn-quiz)

## 🛠 주요 개발 마일스톤

### 1. 데이터 파이프라인 구축
- **수집**: 나무위키 및 미러 사이트의 캐릭터 페이지 145개를 자동으로 크롤링하여 로컬 HTML 저장.
- **가공**: `scripts/build_dataset.mjs`를 통해 HTML 내의 인포박스(마법, 좋아하는 것 등) 및 개요 섹션을 정교하게 파싱하여 `characters.json` 생성.
- **최적화**: 수집된 이미지를 WebP 포맷으로 변환 및 최적화하여 145명 전체 이미지 용량을 1.8MB로 압축.

### 2. 퀴즈 엔진 고도화
- **오답 생성 로직**: 캐릭터의 실루엣(`silhouette`), 이름 유사도(접두사/접미사), 기수 정보를 가중치로 계산하여 난이도별 오답 구성.
- **상태 관리**: Vanilla JS 기반의 전역 `state` 객체를 활용한 화면 전환 및 정답 관리 로직 구현.

### 3. UI/UX 디자인 (Teenieping Bright Pink Theme)
- **테마 적용**: Apple Design 가이드를 기반으로 시작했으나, 사용자 피드백을 반영하여 티니핑 특유의 화사한 핑크톤 및 애니메이션 효과가 강조된 독자적 테마로 전면 개편.
- **반응형 최적화**: 
  - 모바일과 PC 모두에서 스크롤 없이 한눈에 들어오는 **100vh(No-Scroll)** 레이아웃 구현.
  - 디바이스 높이에 따른 요소 크기 자동 스케일링 적용.

### 4. 기술적 문제 해결
- **모바일 오디오 이슈**: iOS/Android 브라우저의 오디오 자동 재생 차단 정책을 해결하기 위해 첫 사용자 제스처 시 `AudioContext`를 비동기적으로 활성화(Unlocking)하는 로직 적용.
- **이미지 사이즈 불균형**: `object-fit: contain`과 `min-height: 0`을 조합하여 다양한 비율의 이미지가 컨테이너 내에서 완벽하게 정렬되도록 수정.

## 📁 주요 파일 구조
- `web/`: 실제 서비스되는 프론트엔드 코드
  - `index.html`: 메인 진입점
  - `styles.css`: 티니핑 테마 스타일시트
  - `app.js`: 퀴즈 로직 및 상태 관리
- `scripts/`: 데이터 수집 및 가공 도구들
- `data/`: 원본 및 가공된 캐릭터 데이터
